# 0001. 与上游 pshenok/server-survival 彻底分家

日期:2026-07-17
状态:已接受

## 背景

本仓库是 pshenok/server-survival 的 fork,此前 main 与上游保持同步。上游仍在活跃开发(近期集中于 bugfix 与数值平衡)。

本项目的目标是做移动端 Web 版并最终移植为微信小游戏,这要求:拆分 4000 行的 game.js、引入 Vite 构建、重组目录、建立平台适配层。这些改动会使后续的上游 merge 在实践中不可行——这是一个单向门。

## 决策

接受与上游的永久分叉,把本仓库当作独立项目维护:

1. 重构动工前,最后一次手动同步上游 main 作为基线。
2. 之后上游的改动只做**选择性人工搬运**(逐个 bugfix/数值调整对照移植),不再尝试 git merge。
3. 不以"贡献回上游"为目标约束本仓库的技术决策。

## 备选方案

- **先观察,等上游节奏放缓再重构**:被否决——窗口期不可预测,移动端目标会被无限期推迟。
- **双轨维护(跟随上游的 main + 长期重构分支)**:被否决——对个人项目维护成本过高。

## 后果

- 收益:技术决策完全自主,重构无需顾虑 merge 冲突。
- 代价:失去上游后续 bugfix 的免费搭车,数值平衡改动需人工跟踪(可订阅上游 release/commit 作为提醒)。
- README 中指向上游 GitHub Pages 的"PLAY NOW"链接与品牌信息需在适当时机更新为本仓库自己的部署。
