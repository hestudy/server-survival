# CONTEXT

本仓库的统一语言词汇表。只收录术语定义,不收录实现细节。

## 核心玩法术语(现有)

- **服务 (Service)**:玩家放置在网格上的基础设施节点(防火墙、负载均衡、数据库等),有造价、容量、维护费和健康度。
- **请求 (Request)**:沿连线移动的流量单元,共 6 种流量类型(STATIC / READ / WRITE / UPLOAD / SEARCH / MALICIOUS)。
- **连线 (Connection)**:两个服务之间的有向边,决定请求的路由路径。方向有语义:上游 → 下游。
- **声誉 (Reputation)**:0–100 的全局指标。请求失败、攻击漏过会扣减;归零则游戏结束。
- **维护费 (Upkeep)**:服务按分钟扣除的运行成本,随游戏时长递增(1x → 2x)。
- **限流 (Throttle)**:API Gateway 对超额流量的软失败处理,声誉损失小于硬失败。
- **流量模式切换 (Traffic Shift)**:定期改变各流量类型占比的事件。
- **恶意波次 (Malicious Spike / DDoS Spike)**:周期性的高比例恶意流量攻击波。

## 架构分层术语(模块化引入)

- **仿真核心 (Simulation Core)**:纯游戏逻辑——路由、经济结算、事件系统、健康度、存档序列化。不依赖 DOM、渲染器和任何平台 API,随机性通过注入的 RNG 提供。
- **平台适配层 (Platform Adapter)**:隔离平台差异的边界层。存储、音频、输入事件源只允许在这一层出现,Web 与微信小游戏各有一套实现。
- **渲染层 (Renderer)**:Three.js 场景管理,消费仿真核心的状态,不反向持有游戏逻辑。

## 触屏交互术语(移动端引入)

- **点选 (Tap)**:单指轻点。点服务 = 选中并查看详情;连线工具下先后点选两个服务 = 建立连线。
- **抬起 (Lift)**:长按(约 350ms)服务后进入的可拖动状态,有触觉/视觉反馈。只有抬起后的服务才能被拖动换位;未抬起时单指拖拽一律是相机平移。
- **平移 (Pan)**:移动相机视角的手势——触屏为未抬起状态下的单指拖拽,桌面为右键/中键拖拽。任何情况下不移动服务。
- **捏合 (Pinch)**:双指开合缩放视角,等价于桌面滚轮缩放;双指中点的移动同时平移相机。捏合过程中不会误触点选或单指平移。
- **顶部状态条 (Top Strip)**:小屏上唯一常驻的顶部 UI,承载资金、声誉、时间、RPS 与事件指示。
- **底部抽屉 (Bottom Sheet)**:小屏上按需弹出的标签页容器,承载财务、健康、服务详情等原桌面角落面板的内容。
- **桌面冻结 (Desktop Freeze)**:移动端改造期间的纪律——宽屏断点下的布局与交互保持现状,作为每个重构变更的回归基准。

## 性能术语(性能基线引入)

- **粒子上限 (Particle Cap)**:同时拥有独立 mesh 的请求数量上限。只作用于渲染层——超出上限的请求照常仿真,视觉上并入聚合表现;上限收紧不驱逐存量粒子。
- **聚合表现 (Aggregated Representation)**:超出粒子上限的流量的集体视觉形态——按数量驱动互联网节点光环的脉冲,而非逐请求渲染。
- **降阶档位 (Quality Tier)**:自动降阶阶梯上的当前画质档(pixelRatio 上限 / 阴影 / 材质精简 / 粒子上限的组合)。帧率持续偏低时降档、恢复后回升,当前档位记录在运行状态中。
